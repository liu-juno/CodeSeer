# 列表分页展示 — 设计文档

**需求**: 项目管理、迭代管理、需求管理列表分页展示  
**状态**: 已批准  
**版本**: 1.0  
**日期**: 2026-06-21

---

## 1. 概述

为项目管理、迭代管理、需求管理三个列表页面添加后端分页支持，采用基于偏移量的分页方案（`?page=1&page_size=20`），返回统一分页响应结构 `{ items: [], total, page, page_size }`。前端封装 `usePagination` composable，复用 Element Plus `el-pagination` 组件。

---

## 2. 后端改动

### 2.1 统一分页响应结构

```python
class PaginatedResponse(BaseModel):
    items: List[Any]
    total: int
    page: int
    page_size: int
```

### 2.2 涉及接口

| 接口 | 改动 |
|------|------|
| `GET /projects` | 添加 `page`、`page_size` query 参数，返回 `PaginatedResponse[ProjectResponse]` |
| `GET /iterations` | 添加 `page`、`page_size` query 参数，返回 `PaginatedResponse[IterationResponse]` |
| `GET /requirements` | 已有 `page`、`page_size` 参数（暂不支持），统一改造 |

**分页默认值**: `page=1`, `page_size=20`

### 2.3 响应格式示例

```json
{
  "items": [/* Project objects */],
  "total": 45,
  "page": 1,
  "page_size": 20
}
```

## 3. 前端改动

### 3.1 `usePagination` Composable

```typescript
// src/composables/usePagination.ts
export function usePagination<T>(fetcher: (page: number, pageSize: number) => Promise<{ items: T[], total: number }>) {
  const items = ref<T[]>([])
  const total = ref(0)
  const page = ref(1)
  const pageSize = ref(20)
  const loading = ref(false)

  const fetchPage = async (p: number) => {
    loading.value = true
    try {
      const res = await fetcher(p, pageSize.value)
      items.value = res.items
      total.value = res.total
      page.value = p
    } finally {
      loading.value = false
    }
  }

  const onPageChange = (p: number) => fetchPage(p)
  const onSizeChange = (size: number) => { pageSize.value = size; fetchPage(1) }

  return { items, total, page, pageSize, loading, fetchPage, onPageChange, onSizeChange }
}
```

### 3.2 页面改造

三个页面（Projects、Iterations、Requirements）统一改造：

1. **移除**客户端 computed 过滤逻辑中的分页相关代码
2. **移除** `displayedRequirements` 等前端分页 computed
3. **引入** `usePagination` composable
4. **添加** `el-pagination` 组件，包含：背景色、页码按钮、每页条数选择器、跳转输入框
5. **保留**搜索/状态筛选功能，筛选时重置到第一页

```vue
<el-pagination
  background
  layout="total, sizes, prev, pager, next, jumper"
  :total="total"
  :page-size="pageSize"
  :current-page="page"
  @size-change="onSizeChange"
  @current-change="onPageChange"
/>
```

## 4. 数据流

```
用户操作（翻页/筛选）
    ↓
前端调用 fetcher(page, pageSize)
    ↓
后端 API: GET /projects?page=1&page_size=20
    ↓
数据库查询: SELECT ... LIMIT 20 OFFSET 0 + COUNT(*)
    ↓
返回 { items: [...], total: 45, page: 1, page_size: 20 }
    ↓
前端更新 items、total，el-pagination 自动渲染
```

---

## 5. 搜索/筛选行为

- 搜索框/筛选器触发时：**重置 page = 1**，重新请求
- 后端需支持同时传筛选参数 + 分页参数
