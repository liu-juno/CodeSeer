# UI/Layout Refactor Design

**Date:** 2026-06-27  
**Status:** Approved

## Overview

完整重构 CodeSeer 的 UI 布局，目标风格为 TAPD/飞书/Jira 专业工具感：高信息密度、功能优先、色彩用于状态标记。核心变化是从单栏可折叠 sidebar 改为 TAPD 风格双栏固定布局，并引入迭代上下文（Iteration Context）和手动深色模式。

---

## 1. 整体布局结构

三列布局：项目列表轨道（Rail）+ 模块 Sidebar + 主内容区。

```
┌──────┬──────────────┬────────────────────────────────────────────┐
│ Rail │ Module       │ Topbar                                     │
│      │ Sidebar      ├────────────────────────────────────────────┤
│      │              │                                            │
│      │              │         router-view (内容区)               │
│      │              │                                            │
└──────┴──────────────┴────────────────────────────────────────────┘
```

**尺寸（全部用 CSS 变量 + clamp，不写死 px）：**

```css
--rail-width: clamp(40px, 3.5vw, 56px);
--sidebar-width: clamp(180px, 14vw, 220px);
--topbar-height: clamp(48px, 4vh, 60px);
```

小屏（< 1280px）时模块 sidebar 自动收起到图标模式（只显示 icon，宽度降至 `--rail-width`）。

---

## 2. 左侧项目 Rail

固定宽度 `--rail-width`，不可折叠。

**内容（从上到下）：**
- Logo 区：CodeSeer 缩写图标，点击跳转首页/项目选择页
- 项目列表：每个项目用首字母 + 自定义背景色的圆角方块（32×32）表示
  - 当前项目：主色背景高亮
  - 其他项目：灰色，hover 时右侧 tooltip 显示完整项目名
  - 超出可视区域时列表可滚动
- 底部：「+」创建新项目

**背景色：** `--rail-bg`（深于 sidebar，形成视觉层次）

---

## 3. 模块 Sidebar

固定宽度 `--sidebar-width`，不可折叠（信息密度优先）。

**顶部：** 当前项目名（超长截断 + tooltip）

**导航分三组，细分割线隔开，不加 label 文字：**

| 组 | 菜单项 |
|---|---|
| 工作模块 | 概览、需求、缺陷、迭代 |
| 资源 | 文档、成员 |
| 管理 | 设置 |

**选中样式：** 左侧 3px 主色竖线 + `--bg-elevated` 背景  
**字号：** `0.875rem`，行高宽松  
**Hover：** 微妙背景变化，无突兀动画

---

## 4. Topbar

固定高度 `--topbar-height`，分三个区域。

**左侧 — 迭代选择器**
- 显示：`[📅 迭代名  开始日期 ~ 结束日期  ▾]`
- 点击展开下拉，列出该项目全部迭代，支持搜索
- 可选「全部」（不过滤）
- 未选时显示占位文字「选择迭代」
- 进入「文档」「成员」「设置」页面时自动变灰（disabled）

**中间 — 面包屑**
- 显示当前页面路径，例如：`需求管理` 或 `迭代 / Sprint 3 / 需求详情`
- 次要文字色，不抢视觉焦点

**右侧 — 操作区**
- 全局搜索图标（点击后搜索框覆盖面包屑区域）
- 深色模式 toggle（🌙/☀️ 图标）
- 用户头像（点击展开：个人设置 / 退出登录）

---

## 5. 迭代上下文（Iteration Context）

**全局过滤模式：**
- 切换 Topbar 迭代选择器后，需求/缺陷/任务列表自动带 `iteration_id` 参数重新请求
- 选择存入 `projectStore.currentIterationId`，跨页面导航时保持不重置
- 「文档」「成员」「设置」等页面不受影响，迭代选择器显示为 disabled

**迭代详情页（深度操作）：**

```
┌───────────────────────────────────────────────────────────────┐
│ Topbar：[📅 Sprint 3 ▾]   迭代 / Sprint 3      [🔍][🌙][👤] │
├───────────────────────────────────────────────────────────────┤
│  Sprint 3                                                     │
│  2026-06-01 ~ 2026-06-30   进行中   ██████░░░░ 60%           │
├───────────────────────────────────────────────────────────────┤
│  [概览]  [需求]  [缺陷]  [成员]                                │
├───────────────────────────────────────────────────────────────┤
│                  子页面内容                                    │
└───────────────────────────────────────────────────────────────┘
```

- 信息头部：迭代名 + 时间范围 + 状态 tag + 进度条（按需求完成比例计算）
- 二级 Tab：概览 / 需求 / 缺陷 / 成员，下划线高亮样式
- 详情页内的列表天然只显示该迭代数据，不受 Topbar 迭代选择器二次过滤
- 在详情页切换 Topbar 迭代选择器 → 跳转对应迭代详情页

## 6. 深色模式

**切换机制：** 点击 Topbar 右侧 toggle → 在 `<html>` 上加/移除 `data-theme="dark"` → CSS 变量自动切换。用户偏好存 `localStorage`，刷新后保持。

**Color Token 体系：**

| Token | 浅色 | 深色 |
|---|---|---|
| `--bg-base` | `#f4f5f7` | `#1a1b1e` |
| `--bg-surface` | `#ffffff` | `#25262b` |
| `--bg-elevated` | `#ffffff` | `#2c2d33` |
| `--border-default` | `#e4e5e8` | `#373a40` |
| `--text-primary` | `#1f2329` | `#c1c2c5` |
| `--text-secondary` | `#646a73` | `#909296` |
| `--color-primary` | `#2d5bff` | `#4c7bff` |
| `--sidebar-bg` | `#ffffff` | `#1e1f23` |
| `--rail-bg` | `#f0f1f3` | `#141517` |

所有组件只引用 token，不写死颜色值。Element Plus 主题色跟随 `--el-color-primary`。

---

## 7. 响应式策略

- 所有布局尺寸用 `clamp()` + CSS 变量，不写死 px
- 断点 `< 1280px`：模块 sidebar 自动收起为图标模式
- 断点 `< 960px`：Rail 隐藏，通过汉堡菜单触发
- 内容区字号、间距全部走 CSS 变量
